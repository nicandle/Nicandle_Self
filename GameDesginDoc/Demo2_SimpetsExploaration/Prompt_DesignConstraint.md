# 🧠 脑暴提示词：SimPets 物理派对玩法 (Design Constraints)

> **用途**：将此 Prompt 发送给 AI，用于脑暴符合“单人物理派对”核心体验的具体玩法原型。

## 1. 核心体验锚点 (Core Experience Anchors)
*   **物理质感**：参考《Party Animals》或《Human: Fall Flat》。角色是**笨拙的 (Clumsy)**、**软体的 (Ragdoll)**、**带有惯性的**。
*   **乐趣来源**：
    *   **预期违背**：想走直线却走歪，想拿稳却滑脱。
    *   **混乱喜剧**：与物理环境对抗时的滑稽表演。
*   **核心动作**：**物理交互 (Physics Interaction)**。
    *   **不限于搬运**。可以是抓取、收集、清洁、甚至只是在灾难中存活。

## 2. 设计硬约束 (Hard Constraints)
*   **模式**：**单人 PVE**（禁止设计多人PVP或联机合作）。
*   **规模**：**小品级**。单局时长 **3-5 分钟**。
*   **开发成本**：**低**。
    *   避免复杂的关卡地形设计（推荐模块化或单一场景）。
    *   避免复杂的敌方 AI（敌人应是物理规则本身，如重力、风力）。
*   **上手难度**：**极低**。操作必须简化，适合直觉游玩。

## 3. 游戏循环与目标 (Loop & Goal)
*   **目的**：这是 SimPets (AI宠物游戏) 的**资源产出副本**。
*   **奖励逻辑**：**所见即所得 (What You Get Is What You Keep)**。
    *   **核心概念**：你在关卡里**搞定**了什么，回家就**拥有**什么。
    *   *例1*：把猫爬架塞进车里 -> 获得猫爬架。
    *   *例2*：把脏兮兮的玩具洗干净 -> 获得该玩具。
    *   *例3*：在金币雨中抢到多少 -> 获得多少金币。

## 4. 脑暴指令 (Brainstorming Instructions)
请基于以上约束，构思具体的玩法原型。每个方案需包含：
1.  **一句话概念**：用“在XX环境下做XX事”来描述。
2.  **核心机制**：玩家具体操作什么？挑战是什么？
3.  **所见即所得**：具体描述“在关卡里做了什么”对应“回家得到了什么”。
4.  **如何省成本**：为什么这个玩法开发量小？
5.  **复玩性来源**：靠什么让玩家愿意玩 20 次？
