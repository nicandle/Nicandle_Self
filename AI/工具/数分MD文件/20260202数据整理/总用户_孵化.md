# 数据漏斗需求描述

## 1. 基础信息
- **SQL文件名称**: 
- **数据库**: `nvwa_cbt1`
- **引擎要求**: Spark SQL

## 2. 核心需求
- 分别统计29/30日注册用户数
- 分别计29/30日注册且生涯完成1次孵化的用户数


## 3. 字段信息
- 字段lock_region，参数为 ID 或者 BR
- 日志AccountRegister，local_dt字段注册日期
- 日志AccountRegister，account_id是唯一的，当日有多少ID，就有多少用户
- 单个用户PetClaim日志数量表示为孵化领取成功次数（判断1次即为有这个日志）


## 4. 期望输出字段
- 分别展示29/30日统计注册用户数
- 分别展示29/30日这些用户中生涯完成1次孵化的用户数
- 分ID/BR不同地区展示
