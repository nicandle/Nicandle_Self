# 阶段2：伙伴入住 (Moving In)

## 1. 核心体验 (Core Experience)
**“从蓝图到现实”**
通过基因组合，先在图纸上通过 AI 构想宠物的模样（2D 预览），确认满意后，再投入资源将其“实体化”为游戏内的 3D 伙伴。
*   **体验目标**：赋予玩家“造物主”的仪式感，同时提供低成本的试错乐趣。

## 2. 玩法流程 (Gameplay Flow)

```mermaid
graph LR
    Select[**基因选择**<br/>投入已解锁基因] --> Simulate[**模拟构想 (2D)**<br/>生成概念图]
    Simulate --> Check{**满意吗?**}
    Check -->|不满意| Retry[**微调/重随**<br/>消耗少量生物质]
    Check -->|满意| Realize[**实体化 (3D)**<br/>生成3D模型]
    Realize --> Arrive[**伙伴入住**<br/>生成性格/属性]
    Retry --> Simulate
```

---

## 3. 核心机制 (Key Mechanics)

### 3.1 基因模拟 (2D Preview)
*   **原料投入**：选择 **主基因**（决定骨架/种族）和 **修饰基因**（决定特征/材质）。
*   **AI 绘图**：系统根据基因组合，生成一张 **拍立得风格** 的 2D 概念图。
*   **低成本重随**：如果不满意生成的图片，可以消耗极少量资源进行“重绘”，直到玩家对宠物的长相满意为止。

### 3.2 实体化 (3D Realization)
*   **确认打印**：当玩家确认满意 2D 预览图后，消耗大量资源，将这张图片转化为游戏内的 3D 实体。
*   **特征映射**：游戏将 2D 图片中的关键特征（如颜色、纹理、角、翅膀）映射到 3D 模型上。

### 3.3 居民身份
*   **命名与档案**：宠物诞生后，玩家为其取名。系统会自动生成一张包含出生照（刚才的2D图）、属性、性格的“居民身份证”。

---

## 4. 性格标签规则

系统基于 **2D 预览图的视觉特征** 分析并赋予性格标签，这些标签将决定宠物后续的行为模式。

### 4.1 视觉与性格的关联
| 视觉特征 | 可能生成的性格 | 游戏内行为影响 |
| :--- | :--- | :--- |
| **大眼睛** | [好奇宝宝] | 喜欢盯着新家具看，追随光标。 |
| **尖牙/大嘴** | [贪吃] | 进食频率高，容易饿，但产出谢礼多。 |
| **眯眯眼/睡眼** | [懒癌] / [宅] | 移动速度慢，喜欢像液体一样躺着。 |
| **强壮四肢** | [多动症] | 喜欢跑酷 (Zoomies)，互动动作幅度大。 |

### 4.2 标签分类
*   **生活习性**：如 [夜行性]（晚上活跃）、[洁癖]（主动清理垃圾）。
*   **社交偏好**：如 [社牛]（主动找人玩）、[独行侠]（喜欢角落）。
*   **特殊怪癖**：如 [破坏王]（弄坏家具）、[寻宝猎人]（在家挖宝）。
