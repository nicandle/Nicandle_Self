# 阶段3：家园建设 (Homeland)

## 1. 核心体验 (Core Experience)
**“为你的伙伴筑巢”**
建设的重心不再是单纯的堆砌数值，而是**观察宠物的需求**，为它们打造真正舒适的栖息地。
*   **体验目标**：从“为了刷分而摆家具”转变为“为了看宠物开心而摆家具”。

## 2. 玩法流程 (Gameplay Flow)

```mermaid
graph LR
    Observe[**观察需求**<br/>宠物想要什么?] --> Build[**布置家园**<br/>放置对应家具]
    Build --> React[**宠物反馈**<br/>互动/使用家具]
    React --> Mood[**心情提升**<br/>产出谢礼]
    Mood --> Unlock[**解锁图纸**<br/>更多专属家具]
```

---

## 3. 核心机制 (Key Mechanics)

### 3.1 宠物中心主义
*   **需求气泡**：宠物头顶会直接冒出图标表达需求（例如：🔥图标表示冷，想要暖炉；🍖图标表示饿）。
*   **家具互动**：所有功能性家具必须**可被使用**。
    *   *例子*：猫抓板（抓挠）、温泉池（泡澡）、纸箱（钻入）。
*   **观察反馈**：玩家的核心乐趣在于观察宠物如何使用这些家具。

### 3.2 标签共鸣规则
*   **范围力场**：家具会产生**范围力场**（例如：暖炉半径3格为[温暖区]）。
*   **适宜度判定**：只要宠物处于符合其性格标签的区域（如[火系]在[温暖区]），心情即提升。
*   **拒绝堆砌**：相同类型的力场（如[温暖]）叠加只会扩大范围，不会无限叠加数值。一个房间放一个暖炉即可满足需求。

### 3.3 装饰经济
*   **谢礼机制**：心情好的宠物会定期在窝边留下“谢礼”（资源包）。
*   **门票机制**：美观度高的家园会吸引 NPC 访客，访客参观后会支付门票。

---

## 4. 建造规则

### 4.1 放置规则
*   **网格吸附**：家具自动吸附于地面网格，方便对齐。
*   **力场可视化**：在装修模式下，选中家具会显示其影响范围（如红圈代表温暖，蓝圈代表寒冷）。

### 4.2 评分体系
*   **美观度**：由家具的基础分和风格统一性决定（给玩家看的，影响访客）。
*   **舒适度**：由是否满足宠物需求决定（给宠物用的，影响心情）。
